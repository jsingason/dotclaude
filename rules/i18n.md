# Internationalization (i18n)

**Never hardcode user-visible text.** Always use the translation system.

## Source of Truth

<!-- setupdotclaude: source file + generate command if generated, else remove section -->
<!-- Source: e.g. translations.csv -->
<!-- Command: e.g. npm run generate:i18n -->

Locale files (JSON/TS/PO/etc.) are **auto-generated** — never edit by hand. Edit source → run command → commit both.

## Adding Text

1. Pick a key (follow convention below)
2. CSV/generated: add row to source file, run generate command
3. Otherwise: add key-value to **all** active locale files
4. Use key in code, never raw string

```ts
// ❌ <button>Save changes</button>
// ✅ <button>{t('common.saveChanges')}</button>
```

Missing translation → placeholder: `"[TODO] Save changes"`

## Key Naming

<!-- setupdotclaude: mark active convention -->

| Convention | Example |
|---|---|
| dot-namespace | `user.profile.title` |
| camelCase flat | `userProfileTitle` |
| SCREAMING_SNAKE | `USER_PROFILE_TITLE` |
| kebab-namespace | `user.profile-title` |

Semantic keys (what it means), not presentational. Group by feature/component.

## Formats

| Format | Files | Notes |
|---|---|---|
| JSON | `en.json` | Nested or flat |
| YAML | `en.yml` | Same rules |
| PO/POT | `*.po` | `msgid`/`msgstr` |
| TypeScript | `en.ts` | Type-checked keys |
| CSV (source) | `translations.csv` | **Edit this, not output** |

## Plurals & Interpolation

Use library syntax — no ternary hacks.

```ts
// ❌ t('items') + ': ' + count
// ✅ t('items.count', { count })  → "{{count}} items" / "{count, plural, one {# item} other {# items}}"
```

## Scope

Translate: button/form labels, placeholders, errors, titles, tooltips, ARIA labels, emails.
Skip: logs, dev errors, code identifiers, URLs, date/number formats.
